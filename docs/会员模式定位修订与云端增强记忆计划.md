# 会员模式定位修订与云端增强记忆计划

## Summary

- 把会员模式重新定义为“托管增强记忆模式”，不是“云备份模式”。同步、备份、跨设备只是配套价值，核心价值是云端选择性 embedding 带来的更强 recall。
- 保持 OSS core 边界不变：`createGoodMemory(config)` 和 `GoodMemoryConfig` 不回填订阅、登录、embedding 配置。
- 采用 `hosted-first`：会员是官方首个打包好的增强模式；未来 self-hosted / provider-backed 路径可获得相近能力，但不改这期边界。
- 这期会员增强只落在 `selective embedding + hybrid recall/rerank`，不把 LLM-assisted extraction / consolidation 一起塞进会员承诺。

## Key Changes

- 更新产品与路线文档：
  - 改写 [会员模式与托管记忆方案.md](/Users/hjqcan/Documents/GoodMomery/docs/会员模式与托管记忆方案.md)、[GoodMemory-PRD.md](/Users/hjqcan/Documents/GoodMomery/docs/GoodMemory-PRD.md)、[13-phase-12-provider-layer-embedding-and-router.txt](/Users/hjqcan/Documents/GoodMomery/task-board/13-phase-12-provider-layer-embedding-and-router.txt)，明确“会员更强”来自云端增强记忆，不是备份附属品。
  - 把现有“会员只是增加云端存储/备份”的措辞，改成“会员在本地模式之上增加云端增强记忆能力；同步/备份是支撑能力”。
  - 在 phase 12 中把会员增强收敛到 hybrid recall，LLM-assisted write path 保持后续阶段。
- 明确读写架构：
  - 本地仍是主路径：`remember/recall/buildContext` 先保证 local-only 可用。
  - 会员写入仍是“本地先成功，再异步同步到云端”。
  - 云端收到 durable memory 后，先跑 `embedding eligibility` 决策，只对合格候选生成 embedding 并进入托管向量检索。
  - 会员 recall 流程固定为：本地 rules recall -> 云端 semantic recall（可选）-> explainable merge/rerank -> 现有 context builder。
  - 云端失败、超时、掉订阅时，自动降级为 local-only，不阻塞热路径。
- 限定增强范围：
  - 这期“更强记忆”只指 recall 质量提升。
  - 不把写入侧 AI 提取、AI 合并、AI 冲突解决纳入会员定义。

## Public Interfaces / Types

- `createGoodMemory(config)`、`GoodMemoryConfig` 保持不变；不重新加入 `embedding`、`llm`、`billing` 字段。
- 会员能力继续走产品层子模块，例如 `goodmemory/member`：
  - 保留现有 `auth`、`sync`、`status` 设计。
  - 增加 member recall 元数据：`mode: "local" | "hybrid_recall"`、`degraded`、`cloudSemanticHits`、`eligibilityPolicyVersion`。
- 新增云端选择性 embedding 的策略契约，不在这期锁死最终阈值：
  - `EmbeddingEligibilityContext`：`memoryType`、`scope`、`sourceMethod`、`explicitness`、`lifecycle`、`contentHash/length`、`superseded/correction` 状态。
  - `EmbeddingEligibilityDecision`：`eligible`、`reason`、`priority`、`retryAt?`。
- 这期默认的 semantic candidate universe 先限定为 `fact`、`reference`、`episode`；`profile`、`preference`、`feedback` 继续走 rules-first，不进入会员云端语义增强主链路。

## Test Plan

- 文档一致性：
  - 会员文档、PRD、task-board 都统一表述“会员 = 增强记忆 + 同步/备份”，不再写成“备份优先”。
- 行为测试：
  - active member 且云端健康时，recall 能合并本地命中与云端 semantic 命中。
  - 云端不可用或超时时，recall 自动回退本地结果且不报错。
  - lexical 精确命中不会被弱语义邻居挤掉。
  - 订阅过期后停止云端增强，但本地记忆与本地 recall 不受影响。
- 策略测试：
  - 不合格 candidate 不触发 embedding。
  - 合格 candidate 带有可追踪的 eligibility reason。
  - superseded / deleted memory 不会继续参与云端 semantic recall。
  - 跨设备同步后的记录能在云端进入 semantic recall。
- Eval/trace：
  - trace 明确区分 `rules-only` 与 `cloud-semantic` 来源。
  - 增加 eval slice，对比 local rules-only 与 member hybrid recall 的提升幅度。

## Assumptions

- 会员是官方首个增强模式，不等于永久独占全部增强能力。
- “更强”在本期只承诺 recall 增强，不承诺 write-path AI。
- “不是所有数据都 embedding”在本期先落实为策略接口、trace 字段、测试约束和默认候选范围；具体阈值与细则留到下一阶段细化。
