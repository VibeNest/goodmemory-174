the installed-host pre-action unification slice.

Contract
--------
Add these installed CLI surfaces:

```bash
goodmemory codex hook pre-tool-use
goodmemory codex action --session-id <id> --command "<command>" [--action-id <id>] [--turn-id <id>] [--sequence <n>] [--json]
```

The hook reads Codex `PreToolUse` Bash payloads from stdin and either:

- returns no output for allow / allow_with_guidance / fail-open cases
- returns a deny payload that points Codex to the installed action bridge

The action bridge performs:

- `assessAction()`
- `resolveHostActionExecutionPlan()`
- rewrite or veto decisions
- first-step execution when allowed
- host-event lineage and evidence on the installed storage path
