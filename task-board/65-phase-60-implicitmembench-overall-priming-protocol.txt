Phase 60: ImplicitMemBench Overall And Priming Protocol
======================================================

Status
------
Accepted / closed for deterministic protocol.

The accepted Phase 60 quality gate is
`reports/quality-gates/phase-60/run-20260505120000/phase-60-quality-gate.json`.
The remaining full-300 Postgres-backed rerun is research follow-up evidence, not
part of the deterministic protocol gate.

Scope
-----
Phase 60 upgrades the ImplicitMemBench research protocol so GoodMemory can make
an official-comparable full-300 statement instead of only a blocking
`procedural_memory + classical_conditioning` statement.

The trigger is the post-Phase-58 full-300 summary:

- `goodmemory-raw-experience`: `90 / 200` blocking at the original trigger
- `goodmemory-distilled-feedback`: `151 / 200` blocking at the original trigger
- priming is present for raw, but not yet a first-class comparable overall lane
- distilled currently omits priming by contract, so `151 / 200` is not an
  official full-300 overall score

Phase 60 makes priming explicit, scored, contamination-checked, and aggregated
into named full-300 overall metrics without reinterpreting historical Phase 49
blocking evidence.

Workstreams
-----------
1. Overall score contract:
   - accepted: define `blockingScore`, `primingScore`, and
     `full300OverallScore`
   - accepted: record denominator, weighting, normalization, and official-comparability
     status in every report
   - accepted: keep historical `passedBlockingCases / totalBlockingCases` unchanged

2. Controlled priming lane:
   - accepted: add `goodmemory-controlled-priming` and
     `goodmemory-distilled-feedback+controlled-priming` summary profiles
   - accepted: preserve experimental/control branch separation from the Phase 49
     runner
   - accepted: score thematic influence only when the final answer still satisfies the
     probe constraints

3. Contamination and leak guards:
   - accepted: detect explicit priming-source phrase reuse, memory-note leakage, and
     copied nouns that the probe forbids
   - accepted: separate "beneficial implicit influence" from "polluting carryover"
   - accepted: make contaminated priming influence ineligible for positive
     overall credit

4. Runner, summary, and gate upgrades:
   - accepted: add Phase 60 runner scripts without changing accepted Phase 49
     smoke gates
   - accepted: update comparison summaries to show raw, distilled/blocking, controlled
     priming, and official-comparable overall profiles separately
   - accepted: produce machine-readable JSON suitable for later public evidence review

5. Full-300 follow-up and claim boundary:
   - pending follow-up: rerun the five-shard Postgres-backed full-300 protocol
     after the runner is upgraded
   - pending follow-up: update the research summary with both blocking and
     official-comparable overall numbers
   - accepted boundary: keep the result research-only until the protocol,
     contamination checks, and rerun evidence are reviewed

Acceptance Criteria
-------------------
- [x] `bun run typecheck` passes.
- [x] Phase 60 unit/regression suites prove priming is included in the new protocol.
- [x] Legacy Phase 49 smoke/gate behavior remains stable or is deliberately versioned
  so accepted historical evidence is not reinterpreted.
- [x] The new report schema includes:
  - `blockingScore`
  - `primingScore`
  - `full300OverallScore`
  - `overallComparableToOfficial`
  - `primingContaminationCount`
  - `primingTaskViolationCount`
  - `primingExplicitLeakCount`
- [ ] The controlled priming lane covers all 100 priming cases in a full-300 run.
- [x] Contaminated priming cases do not increase the overall score.
- [ ] The full-300 rerun summary can answer:
  - whether any GoodMemory profile exceeds the paper's `66%` overall reference
    line under the upgraded protocol
  - whether GoodMemory improves the base model on the same official-compatible
    denominator
- [x] Full-300 remains internal research evidence, not a release hard gate or public
  product claim.

Non-Goals
---------
- Do not claim official leaderboard placement.
- Do not make full-300 ImplicitMemBench a release hard gate.
- Do not add public API/config or a public durable memory kind.
- Do not optimize for benchmark task filenames, case ids, or per-task prompt
  patches.
- Do not let priming reward explicit source-text copying or task-constraint
  violations.
