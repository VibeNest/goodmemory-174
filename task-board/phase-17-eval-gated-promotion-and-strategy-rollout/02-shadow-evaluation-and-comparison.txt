Phase 17.2: Shadow Evaluation and Comparison
============================================

[DONE] P17.2-T001 Run shadow comparisons without mutating default behavior
[DONE] P17.2-T002 Keep contamination-safe baseline vs candidate traces
[DONE] P17.2-T003 Attribute differences to strategy family and mode

Why this exists
---------------
- Strategy experimentation is only trustworthy if candidate behavior can be observed without silently becoming user-facing truth.
- Shadow runs should tell you what would have happened, not leak into what did happen.

Write tests first
-----------------
- shadow runs do not mutate durable memory or promoted defaults
- comparison artifacts keep baseline and candidate traces distinct
- reports can attribute comparison rows to retrieval/reviewer/maintenance families

Implementation steps
--------------------
1. Add shadow execution plumbing to eval and relevant offline runners
2. Persist candidate outputs separately from baseline artifacts
3. Extend reports with contamination-safe diff metadata

Current implementation slice
----------------------------
1. Persist canonical `shadow-executed-path-comparisons.json` per eval run, with `shadow-comparisons.json` kept as a compatibility alias
2. Add `shadowSummary` into suite reporting
3. Attribute rollout comparisons by strategy family and mode
4. Keep baseline trace and executed trace paths explicit while `observe` mode still executes the promoted strategy
5. Run isolated retrieval shadow replays through a separate memory handle so observe-mode candidate evaluation does not mutate the promoted execution path
6. Persist separate `shadow.json` and `shadow-raw-recall.json` traces for the isolated candidate replay

Phase-17 Closure Note
---------------------
- Retrieval is the only active rollout family in the v1 phase-17 slice, and shadow artifacts now carry explicit family/mode attribution for the active path.
- Reviewer and maintenance rollout generalization is deferred to a future phase until those families have real rollout candidates instead of only internal machinery.

Acceptance
----------
- Shadow evaluation is trustworthy and auditable
- Candidate strategies can be compared before promotion
