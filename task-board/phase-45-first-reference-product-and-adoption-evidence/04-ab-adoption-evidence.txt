Goal
----

Produce a product-facing adoption report that compares GoodMemory against a
no-memory baseline, proves rules-only behavior, and keeps provider-backed
uplift as explicit skipped local evidence until a real runner exists.


Required Modes
--------------

- no-memory baseline
- rules-only GoodMemory
- provider-backed uplift explicitly skipped locally; explicit provider-backed
  requests block until a real runner exists


Metrics
-------

- first useful recall rate
- wrong recall rate
- missed recall rate
- correction success rate
- stale memory rate
- user-visible setup steps
- time-to-first-memory-value
- observe-to-selective conversion readiness

`observe-to-selective conversion readiness` is not a narrative-only metric. It
must report:

- observed candidates reviewed
- observed candidates accepted as useful
- observed candidates rejected as unsafe or noisy
- accepted/reviewed ratio
- number of scenarios where selective writeback would be justified

Acceptance Criteria
-------------------

- The report shows where GoodMemory wins and where it fails.
- Provider-backed uplift is explicit and never confused with rules-only closure.
- The gate fails on fabricated or summary-only reports.
- Metrics are product-readable and linked back to scenario families.
