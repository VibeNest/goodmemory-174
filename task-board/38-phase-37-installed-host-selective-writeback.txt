Phase 37: Installed Host Selective Writeback
============================================

Phase Goal
----------
Move the installed-host line from "enabled hosts can automatically recall" to "enabled hosts can opt into selective automatic remember" for Codex first, without widening the root `GoodMemory` API or persisting raw transcripts.

This phase exists to close the current product loop:

```text
Codex installed host
  -> normal user conversation or coding collaboration
  -> GoodMemory selectively extracts high-value memory at turn/session end
  -> GoodMemory writes durable memory through the public remember surface
  -> the next SessionStart/UserPromptSubmit hook automatically recalls it
  -> no manual goodmemory remember seed is required
```

The product rule for this phase is strict: installed-host writeback extracts auditable, explainable, reversible memory candidates from transcript-like host payloads or host-confirmed events. It does not make transcript storage a memory layer.

Definition of Done
------------------
- Codex installed host supports opt-in writeback mode:
  - `off`
  - `observe`
  - `selective`
- `off` is the default.
- `observe` produces candidates and trace, but writes nothing.
- `selective` writes durable memory only through the accepted public `remember` surface.
- no raw transcript is persisted as durable memory.
- assistant-originated memory is ignored unless host annotation confirms or verifies it and the active profile allows it.
- `remember: "never"` masks content before deterministic, custom, or assisted extraction.
- a two-session Codex scenario works without manual `goodmemory remember`:
  - session 1 creates an open loop
  - writeback stores it
  - session 2 recall injects it
- Phase 35 and Phase 36 gates still pass.
- provider-backed live-memory report exists and is accepted.
- external consumer smoke exists and is accepted.

Scope Boundary
--------------
- This phase is installed-host capability, not a new root `goodmemory/writeback` public module.
- This phase is Codex-first. Claude parity may exist, but Claude is not a live blocker.
- This phase does not add a dashboard.
- This phase does not add managed cloud.
- This phase does not add a Memory OS positioning layer.
- This phase does not add a built-in OneLife preset.
- This phase does not reopen the recall router or retrieval promotion.
- This phase does not make automatic writeback default-on.
- This phase does not save full transcripts.
- This phase does not expand the root public API beyond already accepted Phase 36 remember configuration.
