Phase Goal
----------

Push GoodMemory beyond soft behavioral steering by:

- turning text-response policy into harder enforceable output constraints where
  the product contract warrants it
- continuing procedural anti-exemplar-collapse so symbolic or transferable
  rules survive instead of one remembered answer dominating a family
- recovering exact first-action behavior as a dedicated mechanism instead of
  hoping generic text steering will carry it


Dependencies
------------

- Phase 31 native host outcome and correction closure
- Phase 34 host pre-action policy and veto contract
- Phase 43 Runtime Kit
- Phase 49 full ImplicitMemBench research harness
- Phase 51 typed behavioral memory and enactment hardening
- `docs/GoodMemory-ImplicitMemBench-Full-300-Research-Summary.md`


Accepted Scope
--------------

- Add an internal hard-constraint response layer for typed behavioral policy,
  including:
  - forbidden term / fragment blocking
  - required warning or caution text
  - required safe alternative / replacement target
  - required backup mention where side-effect policy demands it
  - concise response-shape controls when the rule is explicitly
    "just the answer" style
- Upgrade typed behavioral policy compilation so these constraints come from
  general repeated evidence and explicit corrective feedback, not benchmark
  file heuristics.
- Strengthen procedural transfer boundaries so single exemplars remain
  `example_only` and symbolic or transformation-style rules are only compiled
  when evidence is sufficient.
- Add a more explicit exact first-action recovery path for covered host/runtime
  surfaces and strict eval cases.
- Keep the slice internal-first and additive on existing `validated_pattern`
  artifacts, with no new durable public memory kind.


Out Of Scope
------------

- per-benchmark task-file hacks or prompt patches
- widening the root `goodmemory` API or README defaults
- making full ImplicitMemBench a release hard gate
- hosted dashboard, cloud sync, team workspace, or raw transcript archive work
- replacing the accepted Phase 51 evidence instead of building on top of it


Acceptance Criteria
-------------------

1. Typed behavioral policy can express hard text-response constraints and these
   constraints are applied without surfacing raw memory-note phrasing.
2. Covered conditioning regressions that require warning/replacement/backup or
   forbidden-term control improve through general mechanisms rather than task
   special-casing.
3. Covered procedural regressions prove stronger anti-exemplar-collapse and
   better symbolic or transformation-style transfer.
4. Covered first-action regressions improve through a dedicated enactment
   mechanism rather than only text steering.
5. Public API, public config, README defaults, and release hard gates stay
   unchanged.
