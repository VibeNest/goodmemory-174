Phase 12: Provider Layer, Embedding, and Router
==============================================

Phase Goal
----------
Turn `llm`, `embedding`, and `router` from architecture-only concepts into staged, testable product capabilities without breaking the current OSS-first simplicity.


Current Status
--------------
[WIP] Phase 12 has started now that Phase 14 archive/evidence/artifact foundation is complete.


Dependencies
------------
- Phase 9 product eval operational
- Phase 10 CLI/examples/release artifacts present
- Phase 11 public surface aligned with current implementation
- Phase 13 governance controls complete
- Phase 14 archive/evidence/artifact foundation complete


Primary Deliverables
--------------------
- provider-layer plan grounded in Vercel AI SDK
- embedding abstraction and vectorization hooks
- router strategy framework
- hybrid retrieval path built on top of archive/evidence-aware recall
- optional LLM-assisted extraction/reranking
- eval slices that prove when provider-backed features help


Definition of Done
------------------
- `llm`, `embedding`, and `router` are no longer only architecture words
- each new capability has rules-first fallback behavior
- every provider-backed capability has deterministic tests and live eval coverage
- public API remains minimal until the behavior is real

Key Constraints
---------------
- Keep `storage` as the only required runtime config until provider-backed behavior is actually used
- Do not add `llm-only` as the default router
- Keep deterministic fallbacks for CI and offline development
- Route all live model calls through Vercel AI SDK adapters
- Preserve explainability for both write-path and recall-path decisions
- Locale-aware rules-first behavior must stay usable without provider-backed multilingual retrieval
- Do not use this phase to backfill missing archive/evidence/artifact foundations that belong to Phase 14


Exit Checklist
--------------
- Provider abstraction exists
- Embeddings can be turned on without changing app code shape
- Router strategies are inspectable
- Hybrid retrieval has eval evidence
- LLM-assisted features are optional, not mandatory
- Docs and examples explain when to stay rules-only
