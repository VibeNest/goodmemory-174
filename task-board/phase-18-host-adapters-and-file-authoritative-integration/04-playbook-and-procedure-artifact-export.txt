Why this exists
---------------
- Hosts often consume procedural guidance as files, snippets, or playbooks rather than internal records.
- These exports should be derived artifacts, not the procedural source of truth.

Write tests first
-----------------
- exported playbooks are deterministic from the same validated pattern inputs
- playbooks record their source pattern lineage
- playbook regeneration does not mutate procedural truth

Implementation steps
--------------------
1. Compile validated patterns into host-readable playbook files
2. Keep lineage back to the underlying feedback/proposal chain
3. Render minimal skill/prompt snippets only as derived output forms

Acceptance
----------
- Hosts can consume procedural memory through exported artifacts
- Exported files remain projections rather than truth sources


Implementation Notes
--------------------
- active `validated_pattern` feedback now compiles into deterministic `playbooks/*.md` artifacts during `exportMemory()`
- playbook files carry canonical pattern lineage, including the canonical memory id and evidence ids when present
- prompt and skill snippet variants now compile alongside the canonical playbook file as read-only derived outputs
