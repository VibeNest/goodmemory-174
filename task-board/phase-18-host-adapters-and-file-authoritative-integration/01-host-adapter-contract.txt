Why this exists
---------------
- Claude/Codex-style hosts need a stable integration surface that does not pretend Markdown files are the core schema.
- Adapter modes should be explicit so hosts know whether they are reading projections or authoring structured changes.

Write tests first
-----------------

- unsupported write attempts fail fast
- core mode still works without adapter registration

Implementation steps
--------------------
1. Define host adapter interfaces and capability flags
2. Separate read-only compiled artifact access from writable file-authoritative flows
3. Keep file modes behind optional adapter registration rather than core config bloat

Acceptance
----------

- Core truth boundaries remain intact
