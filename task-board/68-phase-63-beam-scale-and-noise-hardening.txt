Current Failure Surface
-----------------------

- BEAM answer capability is not closed: 122/400 answers are still wrong after
  the answer-hardening evidence pack.
- Top repair families: conflict_update 29 (dominant full-recall-clean),
  instruction_following 27 (dominant full-recall-noisy), temporal_order 24
  (dominant full-recall-clean), aggregate_count 15,
  judge_or_expected_answer 10, summarization 9, abstention 7.
- Live-measured repair landed in `src/answer/evidencePack.ts` and the BEAM
  live-slice context builder: optional
  question-type metadata now routes event/temporal questions to timeline
  evidence, contradiction/update/CR questions to current-value framing, and
  summarization questions to summary framing. The pack also adds order
  answer-shape guidance for requested item counts/topics, value-bearing count
  tables, multi-session facet framing, instruction standing/latest constraint
  framing, contradiction guidance that avoids yes/no-only answers, selective
  assistant companion evidence for synthesis/instruction-like questions, and
  narrow answer-operation guardrails. The live proof improved hard-slice
  accuracy to 10/12 and full-run accuracy to 278/400.
- The next live-measured repair lane is conflict/update resolution first, then
  instruction noise budgeting and temporal-order granularity, not another BEAM
  case-specific recall gate.
- Use noise budgeting for instruction_following only if the live slice still
  shows full-recall-noisy failures after the instruction constraint framing.
- Do not resume BEAM case-specific gates until the analyzer shows a recall-side
  reason to do so.

Working Loop
------------

1. Use `run-phase63-beam-live-answer-gap-answer-hardening-current` as the next
   repair source of truth.
2. Patch the highest-value answer-time lane generically, then validate in a
   focused live slice before another full BEAM rerun.
3. Validate the same answer-time behavior against Phase 64 MemoryAgentBench CR
   before treating a BEAM gain as general.
4. Run the full BEAM live measured run only after a focused live gain; gate it
   only if the full run has `executionFailures: 0`.

Do not treat a small live slice, one selector repair, or the 0.695 measured
checkpoint as BEAM performance closure.
