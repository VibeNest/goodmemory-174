Phase 60.2: Controlled Priming Lane
===================================

Goal
----
Make priming a first-class GoodMemory evaluation lane without rewarding
pollution.

Tasks
-----
- [x] Add a GoodMemory priming summary profile for `priming_pair_judge` cases.
- [x] Keep experimental and control scopes separate:
  - control sees no priming source
  - experimental sees the priming source
  - both scopes are cleaned even when one branch fails
- [x] Preserve the final-generation contract:
  final answer receives only the allowed context and the `test_probe`.
- [x] Add a priming result object with:
  - raw judge score
  - normalized credit
  - contamination flags
  - task-compliance flags
  - explicit leak flags
- [x] Add tests for positive controlled priming and polluted priming.
- [ ] Add broader neutral-priming full-run diagnostics after the five-shard
  rerun exists.

Acceptance
----------
- Priming coverage is exactly 100 cases in a full-300 run.
- Priming scores are visible per profile and per case.
- Positive priming credit is impossible when the answer violates the probe
  constraints or visibly copies forbidden priming text.
