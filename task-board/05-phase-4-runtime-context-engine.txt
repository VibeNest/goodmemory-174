Phase 4: Runtime Context Engine
===============================

Phase Goal
----------
Implement the runtime continuity layer that supports active work without polluting durable memory.


Dependencies
------------
- Phase 3 complete


Primary Deliverables
--------------------
- Session buffer behavior
- Working memory behavior
- Session journal behavior
- optional spillover scaffolding


Definition of Done
------------------
- Runtime state can be created, updated, loaded, and compacted in tests
- Session journal exists as a first-class runtime asset


Tasks
-----
[DONE] P4-T001 Implement session lifecycle API
Write tests first:
- startSession initializes clean runtime state
- appendToSession appends messages deterministically
- endSession finalizes runtime boundaries

Implementation steps:
1. Add session service
2. Track timestamps and ids
3. Support multi-session same user

Acceptance:
- Session lifecycle is stable for later replay tests


[DONE] P4-T002 Implement session buffer
Write tests first:
- messages append correctly

- sliding behavior works when limits are hit

Implementation steps:
1. Implement raw message append
2. Implement summary placeholder support
3. Expose read API for runtime recall

Acceptance:
- Runtime buffer can support long-turn conversations


[DONE] P4-T003 Implement working memory
Write tests first:
- currentGoal updates
