Phase 61: Priming Abstraction And Contamination-Safe Output
===========================================================

Status
------
Implementation accepted; post-Phase-61 / Phase 62A recovery full-300 rerun
accepted as the latest canonical research result for this protocol line.

Scope
-----
Phase 61 improves the Phase 60 full-300 research protocol after the first
Postgres-backed Phase 60 run showed `0 / 100` credited priming under the
controlled lane.

This phase keeps ImplicitMemBench as internal research evidence. It does not
add public API/config, does not add a public durable memory kind, and does not
make the benchmark a release gate.

Workstreams
-----------
1. Comparable summary semantics:
   - [x] Make `bestGoodMemoryOverallRate` consider only official-comparable
     full-denominator profiles.
   - [x] Add a separate blocking-only best-rate field so `75%` blocking evidence
     is visible without being confused with full-300 evidence.

2. Structured priming diagnostics:
   - [x] Add priming violation tags for invalid JSON, markdown, extra keys,
     empty candidates, bad candidate shape, copied source nouns, forbidden
     commentary, and explicit recall leakage.
   - [x] Add aggregate violation counts and examples to the Phase 60 summary.

3. Latent priming lane:
   - [x] Build an internal eval-only latent priming packet from each branch's
     priming phase.
   - [x] Preserve experimental/control scope isolation and keep the control
     branch free of the experimental source theme.
   - [x] Use abstract cues plus a source-noun blacklist instead of passing raw
     priming text into the final prompt.

4. Contamination-safe output:
   - [x] Enforce strict JSON candidate output for priming probes that require it.
   - [x] Repair malformed, markdown-wrapped, extra-key, or source-copying
     priming answers before sending them to the priming judge.

5. Operator rerun path:
   - [x] Add one-command 10-shard live full-300 wrapper:
     `bun run eval:phase-61-full300`.
   - [x] Default to `10` shards, shard-level concurrency `6`, and per-shard
     case concurrency `1`; merge shard reports into one final
     official-comparable summary.
   - [x] Default priming timeout is `180000ms` when the env var is absent, and
     `.env` can override it through
     `GOODMEMORY_IMPLICITMEMBENCH_PRIMING_TIMEOUT_MS`.
   - [x] Force Postgres-backed storage through `GOODMEMORY_STORAGE_URL` or
     `GOODMEMORY_TEST_POSTGRES_URL`.

6. Post-Phase-61 recovery handoff:
   - [x] Preserve Phase 61 as the priming abstraction and contamination-safe
     output mechanism.
   - [x] Record Phase 62A as the recovery follow-up for the remaining
     full-300 blockers: distilled blocking regression, empty/weak distilled
     context, and weak semantic-field priming transfer.
   - [x] Keep the latest `71.09%` result labeled as internal research evidence,
     not a release gate or public leaderboard claim.
