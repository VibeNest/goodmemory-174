Phase 47: Provider-Backed Retrieval Rollout And Quality Promotion
=================================================================

Status
------
[TODO] Phase 47 is queued after Phase 45 adoption evidence and Phase 46
memory-quality repair evidence.


Phase Goal
----------
Promote provider-backed retrieval from optional uplift to a controlled product
capability only when evidence shows it improves real memory quality beyond the
rules-only baseline.


Dependencies
------------
- Phase 45 reference-product A/B evidence
- Phase 46 memory-quality repair evidence
- Phase 28 supported local acceleration guarantees
- Phase 39 HTTP bridge
- Phase 40 release-proof/product-eval evidence
- Current provider and embedding runtime controls


Accepted Scope
--------------
- Provider-backed retrieval rollout controls that stay explicit and measurable.
- Quality promotion criteria comparing rules-only and provider-backed behavior.
- Fallback behavior that keeps rules-only GoodMemory usable when providers are
  unavailable or fail.
- Product docs that explain when provider-backed retrieval is worth enabling.
- Gate evidence that proves provider-backed retrieval improves target quality
  without increasing wrong recall or setup fragility.


Out of Scope
------------
- default-on provider-backed retrieval without accepted promotion evidence
- hiding provider failure behind silent quality claims
- hosted dashboard, cloud sync, or account system
- root public API widening
- viewer mutation routes
- new reference product scope


Acceptance Criteria
-------------------
1. Rules-only remains an accepted, documented product mode.
2. Provider-backed retrieval has explicit enablement and diagnostics.
3. Promotion evidence compares against Phase 45/46 rules-only behavior.
4. Provider failure is visible and falls back safely.
5. The gate blocks promotion if provider-backed retrieval improves recall volume
   but increases wrong recall, stale recall, or setup fragility.
