Phase 43.3: Lifecycle Orchestration
===================================

Status
------
[DONE] Accepted as part of the Phase 43 Runtime Kit closure.

Goal
----
Implement lifecycle methods over existing GoodMemory APIs.

Required Behavior
-----------------
- `sessionStart` initializes runtime state without transcript persistence.
- `beforeModelCall` can render fragment or progressive context.
- `afterModelCall` creates redacted candidates, trace spans, and bounded jobs.
- `sessionEnd` can summarize runtime state under accepted archive rules.
- `observeToolResult` records bounded tool outcome signals only.

Acceptance
----------
- Runtime-kit produces traceable events.
- Runtime-kit has no hidden durable writes outside accepted policies.
