Phase Goal
----------

Run ImplicitMemBench through GoodMemory itself rather than only through
upstream prompt injection, while keeping the full 300-item benchmark as
research evidence instead of a release hard gate.


Dependencies
------------

- Phase 24 implicit behavioral adaptation eval harness
- Phase 25 Layer D behavioral adaptation closure
- Phase 31 native host outcome and correction closure
- Current live/provider-backed memory stack


Accepted Scope
--------------

- External benchmark-root adapter for the full ImplicitMemBench dataset without
  vendoring the upstream repository.
- Explicit adapter manifest that maps every upstream task file to one scorer
  family.
- Research profiles for:
  - `baseline-upstream-chat`
  - `goodmemory-raw-experience`
  - `goodmemory-distilled-feedback`
- Comparison reporting across:
  - `structured_first_action`
  - `text_behavior_judge`
  - `priming_pair_judge`
- Smoke gate that proves the harness and mirrored subset work from a clean
  checkout.


Out of Scope
------------

- making full ImplicitMemBench a release hard gate
- widening the public `GoodMemory` API or public config surface
- changing README-level default runtime behavior
- introducing an outcome-telemetry profile for the upstream 300-item benchmark
- vendoring the full upstream benchmark repository into GoodMemory


Acceptance Criteria
-------------------

1. Full-benchmark runs require `--benchmark-root` or
   `GOODMEMORY_IMPLICITMEMBENCH_ROOT`; the repo only carries a mirrored smoke
   subset plus attribution.
2. GoodMemory paths replay learning/interference into memory and give the final
   generator only `memoryContext + test_probe`.
3. Priming stays paired `experimental/control` and does not run in the
   distilled-feedback profile.
4. The adapter manifest explicitly routes every upstream task file to a scorer
   family; there is no implicit default routing.
5. `gate:phase-49` proves only harness integrity and smoke comparison coverage,
   not full-300 benchmark quality claims.
