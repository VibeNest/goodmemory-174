Phase 15.3: Rules-Only Reviewer
===============================

[DONE] P15.3-T001 Add rules-only post-turn reviewer baseline

[DONE] P15.3-T003 Keep optional assisted reviewer hooks default-off

Why this exists
---------------
- GoodMemory needs a deterministic review path before any model-assisted reviewer is credible.
- The reviewer should compile suggestions, not silently mutate memory.

Write tests first
-----------------
- reviewer emits proposals for repeated correction, stable pattern, and maintenance-worthy cases
- irrelevant or low-signal turns produce no proposal
- reviewer output records whether a rule or optional model path influenced the result

Implementation steps
--------------------
1. Build a deterministic reviewer over experience, evidence, and recall traces
2. Restrict early proposal types to low-risk, high-signal categories
3. Add assisted-review extension points without changing the default path

Acceptance
----------
- Reviewer can generate stable proposals without provider dependency
- Proposal reasons are inspectable and grounded in trace/evidence inputs


Implemented
-----------
- Added `src/evolution/reviewer.ts` with deterministic rules for `memory_revision`, `procedural_pattern`, and `maintenance_action` proposals
- Reused append-only `ExperienceRecord` telemetry plus scoped feedback/fact state to keep the reviewer explainable and rules-first
- Wired post-turn reviewer execution into `createGoodMemory().remember()` and `createGoodMemory().feedback()` without widening the public API
- Kept optional assisted reviewer extension hooks present but default-off
