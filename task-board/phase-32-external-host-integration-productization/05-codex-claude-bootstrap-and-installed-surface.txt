Phase 32.5: Codex and Claude Bootstrap on the Installed Surface
===============================================================

[DONE] Add installed-package bootstrap commands on the Bun CLI for:
- Codex
- Claude Code
[DONE] Keep bootstrap repo-local and package-boundary-safe:
- no repo-internal imports
- no machine-global hidden dependency on this repo checkout
[DONE] Generate or patch the minimum external host artifacts needed for a canonical setup:
- Codex guidance/config snippets
- Claude Code guidance/config snippets
- public-package wiring examples
[DONE] Add installed-package docs, examples, and consumer smoke for both hosts.

Bootstrap Contract
------------------
- this is the only allowed CLI contract widening in Phase 32
- the bootstrap path must run from the packaged CLI (`bun run goodmemory -- ...`)
- generated guidance must point to public package usage only
- bootstrap operations must be idempotent and marker-based so reruns do not duplicate sections
- bootstrap commands may only generate or patch host config, host instructions, and public-package wiring scaffolds
- bootstrap commands must not implicitly create canonical memory state
- bootstrap commands must not depend on a repo checkout or hidden machine-global prerequisites outside the documented setup
- Codex is the canonical product path; Claude bootstrap exists for parity and external usability, not as a second live blocker

Acceptance
----------
- a fresh Bun consumer can install the packed artifact and bootstrap one Codex workspace end to end
- a fresh Bun consumer can install the packed artifact and bootstrap one Claude workspace end to end
- docs-as-written reflect the installed-package bootstrap flow instead of repo-local-only example execution
- consumer-smoke fixtures prove the bootstrap commands and generated public-package wiring work
- the installed CLI remains memory-first/read-first outside this explicit bootstrap widening

Evidence To Add
---------------
- installed CLI regressions
- package-boundary consumer smoke
- updated reference/setup guides

Implemented
-----------
- `src/bootstrap/hostBootstrap.ts` now owns repo-local, marker-based bootstrap scaffolding for Codex and Claude Code.
- `src/cli.ts` now exposes `goodmemory codex bootstrap` and `goodmemory claude bootstrap` without widening the root package surface beyond this explicit bootstrap path.
- bootstrap writes only:
  - `AGENTS.md` or `CLAUDE.md` marker sections
  - `.goodmemory/bootstrap/<host>-export.mjs` public-package export scripts
- generated export scripts use only `goodmemory` and `goodmemory/host`, export compiled artifacts into `.goodmemory/hosts/<host>/`, and never create canonical memory state on bootstrap.
- installed-package docs now cover both hosts:
  - `docs/GoodMemory-Codex-Handoff-Setup-Guide.md`
  - `docs/GoodMemory-Claude-Code-Setup-Guide.md`
- package-boundary consumer smoke now proves:
  - tarball install
  - bootstrap command execution
  - seed memory through public imports
  - Codex and Claude artifact export through generated scripts
