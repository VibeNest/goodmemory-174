Phase 32.4: Coding-Agent Recall and Context Integration
=======================================================

[DONE] Make the canonical `coding_agent` path consume accepted event-backed procedural patterns together with runtime continuity.
[DONE] Prioritize:
- validated `do` / `don't` / `avoid`
- open loops
- session journal state
- accepted relevant evidence excerpts
[DONE] Keep token pressure bounded and lexical-first selection intact.
[DONE] Keep raw traces and bulky evidence out of the injected context.

Why this exists
---------------
- `docs/开发安排.md` says the next step is not “store more memory”, but make coding-agent recall truly use validated procedural patterns and corrections.
- Phase 31 proved first-action changes internally with native host outcome/correction evidence; Phase 32 must make that usable on the external coding-agent line.

Acceptance
----------
- external event-backed patterns can affect `coding_agent` recall/context on the public path
- `buildContext()` emits compact, action-oriented continuity rather than raw trace dumps
- repeated-correction and procedure-adherence eval families are scored against a frozen dual baseline:
  - pre-Phase-31 public text-only adapter path
  - no-memory baseline
- the event-backed public path must not regress against the frozen pre-Phase-31 public text-only path and must show a clear win over the no-memory baseline
- existing general-chat behavior stays regression-covered and does not inherit coding-agent-only heuristics by accident

Evidence To Add
---------------
- recall/context unit + integration regressions
- deterministic external-host eval slices for continuity, correction avoidance, and procedure adherence

Implemented
-----------
- `src/recall/contextBuilder.ts` now renders `coding_agent` packets with an explicit priority order:
  - procedural memory
  - working memory
  - session journal state
  - accepted evidence excerpts
  - then broader durable memory/profile preference support
- `src/recall/scoring.ts` and `src/recall/selection.ts` now prefer active `validated_pattern` feedback over raw same-rule guidance and dedupe repeated rules before the recall limit is applied.
- `src/recall/evidence.ts` now:
  - prioritizes correction/verification/tool-result evidence ahead of generic conversation excerpts
  - dedupes repeated evidence excerpts before applying the recall limit
  - preserves feedback-linked evidence ids when validated patterns inherit lineage through promotions rather than direct evidence record links
- `src/api/createGoodMemory.ts`, `src/api/evolutionRuntime.ts`, `src/evolution/observations.ts`, and `src/api/agentEventIngestion.ts` now propagate `evidenceIds` through `feedback()` experiences so correction evidence becomes canonical lineage for promoted `validated_pattern` records.
