Phase 17: Eval-Gated Promotion and Strategy Rollout
===================================================

Phase Goal
----------
Close Wave 6 as a retrieval-first rollout and public-surface closure, with reusable lifecycle vocabulary prepared for reviewer and maintenance rollout later.


Dependencies
------------
- Phase 12 provider-layer work complete enough to compare rules-only and provider-backed strategies
- Phase 15 proposal pipeline complete enough to emit decision records
- Phase 16 outcome-aware maintenance available for strategy comparison


Primary Deliverables
--------------------
- retrieval-first `observe -> assist -> promote` strategy modes
- shadow evaluation support and contamination-safe comparison artifacts
- promotion gates and trusted release authorization for retrieval strategy changes
- regression dashboard artifacts and release-facing summaries
- explicit public surface decision criteria for advanced/evolution capabilities and the official OSS CLI shape
- dedicated phase-17 fallback and live-memory quality gates


Definition of Done
------------------
- retrieval candidate strategies can run in shadow before affecting default behavior
- retrieval promotion decisions are backed by eval evidence, persisted records, and an explicit trusted authorization artifact
- rules-only baseline remains first-class and non-regressing
- the decision to expose or keep advanced surfaces internal is evidence-based
- the official OSS CLI shape exists on the supported public path


Tasks
-----
[DONE] P17-T001 Define strategy lifecycle modes: observe, assist, promote
[DONE] P17-T002 Add shadow evaluation and comparison artifacts
[DONE] P17-T003 Add promotion gates for retrieval, reviewer, and maintenance strategies
[DONE] P17-T004 Extend reporting with regression dashboard and gate summaries
[DONE] P17-T005 Decide advanced/public surface boundaries from rollout evidence
[DONE] P17-T006 Update release checklists and operator guidance for promoted strategies


Key Constraints
---------------
- no strategy becomes default without passing an explicit eval gate
- rules-only remains available as a supported baseline
- shadow mode must avoid contaminating user-visible durable state
- rollout evidence must distinguish provider gains from proposal/maintenance gains
- retrieval rollout lands first, and reviewer/maintenance rollout move to a later dedicated phase instead of stretching phase 17 scope
- public-surface decisions must not turn the current eval-artifact inspector into the final OSS CLI story by accident


Exit Checklist
--------------
- Strategy lifecycle modes exist
- Shadow comparison artifacts exist
- Promotion gates exist
- Regression dashboard artifacts exist
- Public surface decision record exists
- Release docs reflect promoted strategy policy

Phase-17 Closure Note
---------------------
- Phase 17 closes retrieval-first on purpose. The lifecycle, shadow, gate, trusted promotion authorization, regression dashboard, dedicated quality gates, and public-surface/CLI machinery are complete for retrieval rollout.
- Advanced runtime salvage controls remain internal and are stripped from the root runtime wrapper, so they are not part of the promoted OSS surface or behavior for retrieval rollout.
- Reviewer and maintenance rollout families move to a later dedicated phase after host adapters instead of remaining implicit unfinished scope inside phase 17.
