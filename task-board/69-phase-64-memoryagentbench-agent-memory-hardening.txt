Phase 64: MemoryAgentBench Agent-Memory Hardening
=================================================

Status
------

[PREP] Phase 64 is the planned MemoryAgentBench leg after Phase 63 BEAM.
This file is a preparation board only: Phase 63 remains active until its BEAM
closure boundary is accepted or explicitly paused.


Driver
------

MemoryAgentBench evaluates memory agents through incremental multi-turn
interactions. Its public README describes four evaluation competencies:
Accurate Retrieval, Test-Time Learning, Long-Range Understanding, and Conflict
Resolution.

Phase 63 already exposed the two issues most likely to block Phase 64:

- broad wrong/noise growth when source-ordered evidence is appended too freely
- knowledge_update regression after recall-lift work

Phase 64 prep therefore starts with readiness analysis, not a benchmark-specific
selector patch.


Accepted Prep Scope
-------------------

- Use the current Phase 63 recall-diagnostic analysis as input.
- Convert Phase 63 residual failures into MemoryAgentBench prep priorities:
  conflict/update handling, noise budgets, behavior-policy learning, and
  long-horizon task state.
- Keep Phase 64 inactive until Phase 63 is closed or explicitly paused.
- Prepare adapter/gate requirements before touching benchmark data.
- Do not vendor upstream MemoryAgentBench data.


Out Of Scope
------------

- marking Phase 64 active
- final public benchmark report
- MemoryAgentBench-specific recall selectors
- README leaderboard claims
- public API/config widening

Current Prep Boundary
---------------------

Phase 64 should not start by adding another source-order lexical branch. The
first useful effect-improvement preparation is to force every future
MemoryAgentBench run to report:

- answer/task accuracy
- evidence recall
- stale or superseded evidence selected
- noisy evidence selected
- action-policy transfer for learned behavior rules

This keeps Phase 64 aligned with GoodMemory's product mechanism instead of
overfitting one benchmark surface.
