Design Notes
------------
Assistant output is not authoritative by default. A life coach may suggest a plan, but the plan becomes durable memory only when the user confirms it, the host verifies it, or the host explicitly tags the turn as durable.

Top-level memory kinds remain fixed:

- profile
- preference
- reference
- fact
- feedback
- episode

Domain semantics live in category, tags, attributes, subject, fact kind, reference kind, and evidence.


Acceptance
----------
- assistant messages remain ignored by default
- annotated/verified assistant messages can write durable memory under an allowing profile policy
- domain metadata survives persistence, export, and trace output
- no recall rendering becomes noisy by dumping arbitrary attributes into user-facing context by default
