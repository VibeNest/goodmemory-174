Design Notes
------------
The DSL should stay small and map to the existing candidate model. It is not a second storage taxonomy.

Example target shape:

```ts
rememberRules.fact(/my top priority this quarter is (.+)/i, {
  id: "life-goal-priority",
  category: "goal",
  tags: ["life_coach", "long_term_goal"],
  content: ({ match }) => match[1],
});
```


Acceptance
----------
- rules produce normal `MemoryCandidate`s
- annotations can suppress and enrich generated candidates
- trace output can explain which rule or annotation affected a write
- policy hooks still run after rule/annotation processing
