Phase 35: Installed Host Memory Middleware and Hooks
====================================================

Phase Goal
----------
Turn the accepted external host line from "repo-local bootstrap plus compiled artifacts" into "global install, repo opt-in, always-on recall middleware" for Codex and Claude Code without widening the root `GoodMemory` API.

This phase exists to make GoodMemory feel closer to a real host-native memory layer through:

- `goodmemory-mcp`
- `goodmemory install codex`
- `goodmemory install claude`
- a thin generated `AGENTS.md` / `CLAUDE.md` rules layer
- `SessionStart` and `UserPromptSubmit` hooks that inject compressed recall from the existing GoodMemory core before the host takes the next step

Definition of Done
------------------
- the canonical product line remains `goodmemory + goodmemory/ai-sdk + goodmemory/host`; no new root `GoodMemory` API is added
- no public `goodmemory/evolution` entrypoint is introduced
- installed-host middleware stays `Hook + MCP` mixed:
  - hooks handle always-on automatic recall injection
  - MCP handles deep read, debug, and artifact browsing
- `goodmemory install codex` and `goodmemory install claude` are idempotent and only write documented user-level config plus repo-local opt-in scaffolds
- repo opt-in is explicit and host-scoped; install alone does not silently activate every repository
- hook handlers use the existing `recall()` + `buildContext()` path and do not invent a second memory engine
- hook handlers fail open:
  - opt-in missing
  - parse/config failure
  - empty memory
  - recall/buildContext failure
  - any of the above returns no injected context rather than blocking the host
- default hook retrieval profile is `coding_agent`
- the installed-host middleware path uses a dedicated user-level SQLite default such as `~/.goodmemory/memory.sqlite` without changing the accepted repo-local CLI default
- explicit write commands are available for durable host-user seeding and correction, but Phase 35 does not add automatic writeback or transcript persistence
- Codex is the only live gate-blocking host path
- Claude Code reaches install/hook/docs/package-smoke parity without becoming a second live blocker
- one canonical Codex live report proves:
  - `SessionStart` continuity injection
  - `UserPromptSubmit` dynamic recall injection
  - installed-package global config plus repo opt-in behavior
  - MCP registration on the installed path

Scope Boundary
--------------
- This phase productizes the accepted external host line; it does not reopen the root API.
- This phase does not turn evidence or hook payloads into a transcript database.
- This phase does not add automatic writeback, automatic maintenance, or a `Stop` hook.
- This phase keeps Phase 34 pre-action veto behavior separate; recall middleware must layer on top of the accepted host boundary instead of replacing it.
- This phase does not make Claude a second live blocker.
