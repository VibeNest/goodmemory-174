longer blocked on finding an embedding endpoint. Representative conv-1 raw
retrieval showed 0.069 recall and 0.92 zero-retrieval, then dialog windows,

BEAM safety checks. The local hashed lexical embedding adapter remains forbidden
for semantic claims because it is another lexical/tie-break proxy.

[P65-R003 RESOLVED 2026-06-27, NEGATIVE — commit bf7373f; banked here 2026-07-02]
The prerequisite was met (OpenRouter /v1/embeddings, text-embedding-3-small,
1536-dim, real vectors) and the probe ran (scripts/measure-locomo-neural.ts,
four arms, deterministic gold-turn recall, no judge, no network beyond the
embedding calls):

  arm                                  recall(40Q)   recall(180Q)
  jaccard baseline (rules-only)        14.2%         8.4%
  BM25 additive (embedding-free)       44.8%         33.7%
  neural text-embedding-3-small        44.8%         33.7%   <- ties BM25 exactly
  hybrid-constant-embed (attribution)  14.2%         -

A real neural embedding adds ZERO recall over embedding-free BM25. The
constant-embed arm proves the lift comes from the additive ranking signal (not
strategy=hybrid itself), and neural != constant proves the endpoint genuinely
works. REFINED CONCLUSION: the ceiling is not the signal source but
CANDIDATE-POOL ADMISSION — semanticScore (and BM25) only re-rank candidates
that already pass the lexical admission gates (hasFactSelectionSignal:
intent/lexical/subject only; the zero-retrieval fallback still needs
lexicalScore >= 0.1), so zero-lexical-overlap gold turns can never be selected
no matter how good the ranking signal is. All measured levers (BM25, neural,
write-time conversational extraction) converge near the same ~45% probe
ceiling. The reopened lane is therefore SEMANTIC CANDIDATE GENERATION (an
opt-in union admission source into fact selection), not an embedding-endpoint
hunt. Do NOT re-run endpoint probes expecting a win.

Driver
------

LoCoMo (snap-research/locomo, "Evaluating Very Long-Term Conversational Memory
of LLM Agents", arXiv:2402.17753) evaluates memory over ten multi-session
conversations between two speakers (avg ~300 turns / ~9K tokens, up to 35
sessions). Each conversation is annotated with question-answering pairs whose
gold evidence is a list of dialog-turn ids (the upstream "dia_id" strings,
formatted "D<session>:<turn>").

QA category codes (verified against snap-research/locomo): 1 multi-hop, 2
temporal, 3 open-domain, 4 single-hop, 5 adversarial. Adversarial samples carry
an "adversarial_answer" (the tempting wrong answer) alongside the correct
"answer". LoCoMo's primary QA metric is token-level F1; adversarial questions
are scored on resisting the tempting answer.

License: CC BY-NC 4.0 (NON-COMMERCIAL). This is stricter than MemoryAgentBench's
MIT, so the no-vendoring discipline is mandatory: no upstream LoCoMo data is
copied into the repo. The external-root convention (GOODMEMORY_LOCOMO_ROOT/
cases.json with already-normalized cases) is the only path that touches real
data.


Accepted Scope (bring-up)
-------------------------

- Mirror the Phase 63 BEAM / Phase 64 MemoryAgentBench recall-diagnostic seam:
  deterministic in-memory storage, hash embedding adapter, deterministic
  createId/now, every dialog turn preserved as a retrievable fact, rules-only
  recall.
- Retrieval-only smoke by default; report per-category evidence-turn recall,
  missing evidence, noise, and a multi_hop cross-session-chain readiness flag.
- Deferred live-answer scaffold only (injectable answerGenerator, answer scored
  deterministically by the upstream match mode). Do NOT wire a real LLM yet.
- No dataset-specific keyword branches as a first repair; no vendoring.

Base-Retrieval Research (eval-only, retrieval-only; NOT a public claim)
----------------------------------------------------------------------

All numbers below are conv-1 (419 turns, 199 questions, all five categories),
rules-only retrieval, executionFailures 0. The fresh reproduced baseline
(run id locomo-baseline-retrieval) is overall evidence recall 0.0687,
zero-retrieval 0.9196, noise 4.45; single_hop recall 0.0571 / zero-retrieval
0.9429; neighbor-beats-gold 0.5382 (the lexically-best turn is usually NOT the
gold turn). The "worth continuing" bar set for these probes: overall recall
>= 0.15, single_hop recall >= 0.15, zero-retrieval <= 0.75, noise <= ~2x.

[DONE] Retrieval-gap analyzer (commit f20d110, `analyze:phase-65-locomo-retrieval-gap`).
  Committed JSON-in/out diagnostic (smoke-report + cases -> retrieval-gap-analysis).
  Quantifies the bottleneck: zero-retrieval 0.92, single_hop worst at 0.94 (proves
  it is BASE retrieval, not multi-hop reasoning), neighbor-beats-gold 0.54/0.79.

[RULED OUT] P65-R001 positional windows + rules-light query expansion (eval-only
  scripts, UNCOMMITTED -- negative results, harness-coupled). Window retrieval:
  contained recall 0.069 -> 0.121 at radius 8 while noise explodes 4.5 -> 14.2 and
  zero-retrieval stays ~0.88. Query expansion (generic conversational clusters, no
  fixture rules): overall recall 0.069 -> 0.073, only multi_hop improved. Positional
  and hand-crafted lexical bridges are insufficient; the gap is open-ended semantic.

[DONE] P65-R002 turn-caption retrieval probe (Step 3, eval-only; committed as the
  Step 3 research slice: `prepare:phase-65-locomo-captioned`,
  scripts/prepare-phase-65-locomo-captioned-root.ts + unit test). It writes a
  captioned external root where each turn keeps its diaId and raw text but gains a
  gpt-5.5 semantic caption (raw + caption), evidence ids unchanged, so any recall
  lift means the SAME gold turn became retrievable. Leakage guard (tested): the
  captioner is given dialog turns ONLY -- never questions, gold answers, or
  evidence ids. Both variants captioned all 419 turns with 0 caption failures.
    turn-only (paraphrase; run id locomo-caption-turn-only-retrieval):
      overall recall 0.0704, zero-retrieval 0.9246, noise 4.20; single_hop 0.0857.
    local-window-2 (+-2 neighbors used only to ground the center caption; run id
    locomo-caption-window2-retrieval):
      overall recall 0.0779, zero-retrieval 0.9196, noise 4.40; single_hop 0.1000.
  RESULT: caption enrichment does NOT materially move exact gold-turn recall. Both
  variants FAIL the bar (overall 0.07-0.08 vs >= 0.15; zero-retrieval 0.92 vs

  CONCLUSION: per the probe's own decision rule (neither variant moves recall

  ranks by meaning, not surface token overlap -- not captions and not more lexical
  rules. No neural embedding endpoint is currently available
  (ai.gurkiai.com serves no embedding model). Live answer was deliberately NOT run
  (retrieval did not improve materially).
