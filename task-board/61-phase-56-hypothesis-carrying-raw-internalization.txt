Phase 56: Hypothesis-Carrying Raw Internalization
===================================================================

Status
------

[DONE] Phase 56 is closed. Core raw support/conflict retrieval, task-hypothesis
formation, and transient execution are implemented, the targeted gate is
accepted, and the required Postgres-backed full-300 follow-up rerun is written
back into the research summary.

Driver
------

- Phase 55 made the raw lane cleaner and more diagnosable, but it still mostly
  stopped at exemplar retrieval.
- The remaining raw misses showed a second gap:
  - retrieval could select the right family of exemplar
  - but the system still failed to convert that support into a probe-specific
    executable hypothesis
- Phase 56 therefore upgrades raw carryover from:
  - exemplar retrieval only
  to:
  - support/conflict retrieval
  - probe-time task hypothesis
  - transient execution for high-confidence raw host-action and symbolic cases


Phase Goal
----------

Turn raw carryover into a probe-conditioned adaptation path by:

- splitting raw support and conflict candidates instead of treating retrieval as
  a single ranked list
- deriving a transient `RawTaskHypothesis` for the current probe
- executing high-confidence host-action and symbolic hypotheses without turning
  them into new durable public memory kinds
- recording typed raw diagnostics so later slices can distinguish:
  - memory miss
  - support conflict
  - hypothesis missing
  - executor unsafe
  - reasoning after correct hypothesis


Dependencies
------------

- Phase 49 full ImplicitMemBench research harness
- Phase 54 exemplar-first raw internalization
- Phase 55 probe-conditioned raw carryover and retrieval calibration
- `docs/GoodMemory-ImplicitMemBench-Full-300-Research-Summary.md`


Accepted Scope
--------------

- add internal support/conflict-aware raw retrieval views
- add internal `RawTaskHypothesis`
- add transient raw execution for:
  - slot rebinding
  - symbolic formula carryover
  - bounded exact host-action recovery
  - guarded decision hints
- keep the slice internal-first and additive on current durable truth
- add Phase 56 deterministic/live targeted eval and gate wiring


Out Of Scope
------------

- public API widening
- new public durable memory kinds
- base-model finetuning
- benchmark-file-specific runtime hacks
- replacing the accepted distilled lane with raw
