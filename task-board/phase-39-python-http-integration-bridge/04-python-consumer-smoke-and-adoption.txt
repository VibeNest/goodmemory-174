Phase 39.4: Python Consumer Smoke and Adoption
==============================================

Goal
----
Prove that a Python/FastAPI service can consume the bridge cleanly.

Scope
-----
- Provide Python/FastAPI request examples or a minimal consumer smoke.
- Show how a consumer session flow calls the bridge before and after the main
  LLM turn; OneLife's `agent-session` is the reference path.
- Show how product-owned state stays product-owned and is merged with
  GoodMemory context at prompt assembly time; OneLife's `life_os_state` is the
  reference case.
- Show that Python/FastAPI remains the authenticated product boundary and that
  the client/mobile app never calls the GoodMemory bridge directly.
- Include a procedural feedback smoke with explicit idempotency and
  source/provenance payload so retries remain consumer-safe.
- Include a targeted correction smoke where Python resolves a visible
  `memoryId` and calls `/memory/revise` without query target resolution.
- Include a proposal/review mapping smoke where consumer outcomes route as:
  accept/edit -> `/memory/revise`, delete -> `/memory/forget`, lock /
  "do-not-remember" -> consumer-only policy with no new bridge mutation.

Acceptance
----------
- The reference docs include Python request examples.
- A smoke path proves the bridge is consumable from outside the repository's
  TypeScript runtime.
- The consumer path does not require client/mobile changes.
- Feedback examples include idempotency and provenance assumptions.
- Export, forget, and revise examples include scoped authorization assumptions.
