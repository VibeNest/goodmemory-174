Phase 42: Progressive Recall Protocol
=====================================

Status
------
[DONE] Accepted as the Phase 42 Progressive Recall Protocol slice.


Intent
------
Turn progressive disclosure into a GoodMemory-owned recall protocol, not an MCP
one-off. MCP, installed-host context injection, runtime-kit, HTTP bridge, and
viewer surfaces must all be able to wrap the same internal service.


Core Boundary
-------------
Progressive recall is an internal runtime service that sits on top of the
existing public memory APIs and storage/export surfaces. It must not widen the
root `goodmemory` API or make MCP the canonical recall transport.


Planned Interfaces
------------------
- `searchRecallIndex(...)`
- `buildRecallTimeline(...)`
- `getProgressiveRecords(...)`
- `renderProgressiveContext(...)`
- `GoodMemoryRecordRef` using `gmrec:v1:${scopeDigest}:${recordKind}:${id}`
- installed-host `contextMode: "fragment" | "progressive"`


Accepted Scope
--------------
- ProgressiveRecallService with shared index, timeline, detail, and render logic
- stable recordRef protocol for durable/runtime/writeback records
- progressive context renderer with token-cost visibility
- MCP adapter tools that wrap the service
- installed-host contextMode config, migration, status, install/enable flags
- fallback to fragment or compact fragment when progressive detail tools are not available
- strict scope digest, redaction, and no-raw-transcript guarantees


Out of Scope
------------
- dashboard or viewer UI
- default-on writeback
- new root `goodmemory/evolution` export
- replacing `recall()` or `buildContext()`
- making MCP required for all installed-host users
- copying implementation from `third-party/claude-mem-main`


Acceptance Criteria
-------------------
1. MCP is only an adapter; shared progressive logic lives in a GoodMemory
   internal service.
2. Index output exposes `scopeDigest` and minimal workspace/project labels, not
   raw user/tenant/workspace/agent/session ids.
3. Detail fetch accepts only recordRef values and denies cross-scope refs.
4. `remember: "never"`, private, sensitive, and session-only content never leaks
   into progressive index or detail output.
5. Installed-host progressive mode is recommended only when MCP is installed and
   usable; otherwise hook context falls back to fragment output.
6. Old installed-host config without `contextMode` migrates to `fragment`.
7. Phase 42 gate proves token-budget benefit without weakening recall quality
   or governance.
