Phase 54: Exemplar-First Raw Internalization
============================================

Status
------

[DONE] Phase 54 is closed with accepted targeted deterministic/live evidence.

Driver
------

- Full-300 GoodMemory research runs showed a stable pattern:
  - the distilled lane could improve through typed-policy and final-surface
    hardening
  - the raw lane remained weak because it still leaned on prose-like memory
    carryover rather than actual experience-conditioned exemplars
- The remaining product problem was no longer "add more rules". It was
  "retrieve, select, and render the right prior episode without polluting the
  current answer with explicit memory phrasing."
- Phase 54 therefore targeted a general raw-path mechanism:
  - derive internal behavioral exemplars from existing truth
  - select them with interference-aware ranking and abstention
  - render minimal carryover examples instead of behavioral instructions
  - keep consolidation local and prototype-bounded instead of turning raw
    history into another prose policy layer


Phase Goal
----------

Push the raw path toward genuine behavior carryover by:

- making episodic exemplars the primary internal carrier for raw behavioral
  adaptation
- adding interference-aware selection and abstention so semantically similar
  but mismatched history does not pollute the current probe
- rendering minimal carryover exemplars instead of default prose steering
- keeping consolidation local and prototype-bounded so token growth stays
  controlled without inventing a new public durable type


Dependencies
------------

- Phase 49 full ImplicitMemBench research harness
- Phase 51 typed behavioral memory and enactment hardening
- Phase 52 structured text-response enactment and guarded policy execution
- Phase 53 surface determinism, escalation routing, and procedural executor
  recovery
- `docs/GoodMemory-ImplicitMemBench-Full-300-Research-Summary.md`


Accepted Scope
--------------

- add an internal `RawBehavioralExemplar` derivation layer sourced from
  existing experiences, archives, host traces, and accepted correction lineage
- add raw-specific candidate generation, lightweight reranking, and explicit
  abstention for low-confidence carryover
- render raw carryover as minimal exemplars rather than `Prefer...` or
  `Avoid...` prose
- add prototype-bounded consolidation summaries and hard-negative generation as
  internal maintenance output
- keep the slice internal-first and additive on current durable truth, with no
  new durable public kind and no public API widening


Out Of Scope
------------

- base-model finetuning
- making full ImplicitMemBench a release hard gate
- replacing the accepted distilled lane with the raw lane
- benchmark-file-specific prompt hacks
- widening public retrieval profiles or public runtime config

-------------------
