===================================================================================

------

and a Postgres-backed full-300 follow-up rerun.

Driver
------

- `docs/GoodMemory-ImplicitMemBench-Full-300-Research-Summary.md`
  now records a post-Phase-52 full-300 GoodMemory rerun where:
  - `goodmemory-raw-experience` remained weak at `31 / 200`

  - storage: explicit Postgres-backed shard run, not default SQLite


Phase Goal
----------

Push GoodMemory past Phase 52's targeted constraint wins by:

- making slot-level rewrite deterministic for URL/path/filetype surfaces
- making forbidden lexical surfaces genuinely blockable instead of just
  discouraged
- turning distrust and side-effect policies into real escalation routing rather
  than softer recommendation text
- adding a stronger procedural executor for exact syntax, exact format, and
  exact symbolic transformation surfaces


Dependencies
------------

- Phase 49 full ImplicitMemBench research harness
- Phase 51 typed behavioral memory and enactment hardening
- Phase 52 hard-constraint enactment and procedural rule transfer
- `docs/GoodMemory-ImplicitMemBench-Full-300-Research-Summary.md`


Accepted Scope
--------------

- Add deterministic slot-level rewrite for covered text-response surfaces,
  including:
  - unsafe path -> safe path redirect
  - insecure URL -> canonical secure URL redirect
  - unsupported filetype -> supported filetype replacement
- Upgrade lexical blocking so covered forbidden terms are removed or replaced
  before final output rather than merely discouraged in steering.
- Add escalation-routing support for distrust and side-effect policies so
  replacement can be:
  - `use specialist X`
  - `warn and refuse default path`
  - `require backup/safe alternative before proceeding`
- Add a procedural executor path for covered exact surfaces:
  - exact first-action or argument order
  - exact signature/closing/format contracts
  - exact symbolic transformation rules that cannot be left to prose steering
- Keep the slice internal-first and additive on `validated_pattern`, with no
  new durable public kind and no public API widening.


Out Of Scope
------------

- benchmark-file-specific hacks
- changing README defaults or public rollout config
- making full ImplicitMemBench a release hard gate
- hosted dashboard, cloud sync, or team workspace work
- replacing the accepted Phase 52 closure evidence instead of building on top
  of it


Acceptance Criteria
-------------------

1. [DONE] Covered directory/path and filetype rewrite cases pass through
   deterministic surface control rather than ambiguous warning text.
2. [DONE] Covered jargon/forbidden-term cases prove hard lexical blocking
   instead of soft avoidance.
3. [DONE] Covered distrust and side-effect cases prove real escalation routing
   or warn/refuse behavior, not generic analysis fallback.
4. [DONE] Covered procedural exact-syntax and exact-format cases improve
   through a dedicated executor path rather than prose steering alone.
5. [DONE] Follow-up full-300 rerun recovers at least the prior post-Phase-51
   distilled high-water mark (`89 / 200`) while keeping explicit recall leaks
   at `0` on the distilled path.
