Phase 67: Public Benchmark Performance and Claim Promotion
=========================================================

Status
------

[ACTIVE] Phase 67 turns GoodMemory's internal benchmark evidence into honest,
reproducible, public-claim-ready scores. The discipline: no internal number
reaches a public README row until it passes the public benchmark claim gate
(P67-A). This phase opens no LoCoMo retrieval work (paused, board 70) and does
not change the Phase 63 BEAM answer-pack code lane (owned by the BEAM workstream);
P67-D is scoped but deferred until that lane is collision-calm or reassigned.


Goal
----

Turn internal benchmark evidence into honest, reproducible, public-claim-ready
scores. Every claimed benchmark must have a claim report with dataset source,
license, command, commit, model, scorer, and baseline; executionFailures 0; no
same-model judge for public semantic-judge claims; fitted AND generalization
recall reported together for BEAM.


Initial Key Finding (P67-A, 2026-06-24)
---------------------------------------

The claim gate's first verdict over all five benchmarks on 2026-06-24: 0
publicly claimable, each blocked for a precise reason.

- LongMemEval (0.908), ImplicitMemBench (0.711), BEAM (0.695): blocked by a
  SAME-MODEL judge (gpt-5.5 answer + gpt-5.5 judge; .env EVAL and JUDGE are both
  gpt-5.5 @ ai.gurkiai.com) plus unverified provenance/commit.
- MemoryAgentBench (CR 0.959 / AR 0.67): cleanest methodology — deterministic
  upstream match-mode scoring, NO judge, so no same-model bias — but blocked by
  incomplete coverage (TTL 0/30, LRU 1/6) and a missing baseline.
- LoCoMo (0.020): a retrieval boundary, not a performance result.

P67-A INFRA CONSTRAINT: there was no independent judge model available for that
first verdict (only gpt-5.5 was served). So the same-model-judge blocker on
LongMemEval/BEAM could not be cleared by a judge swap at that point. The two
viable JUDGE-FREE public-claim routes were:
  1. MemoryAgentBench — complete TTL/LRU coverage + add a baseline (P67-C); it is
     already deterministic and judge-free.
  2. LongMemEval deterministic subset — report only the judge-free methods
     (abstention/contains/exact/expected_alternative/numeric_count), excluding the
     semantic_judge questions, as a separate honest sub-score (P67-B).
Those were the fastest honest paths; the independent-judge path was infra-blocked
like LoCoMo's neural-embedding path.

Current verdict after P67-B/P67-C promotion (2026-07-02): LongMemEval and
MemoryAgentBench are publicly claimable and README-promoted through
`benchmark-claims/*.json` plus `gate:public-benchmark-claim`; ImplicitMemBench,
BEAM, and LoCoMo remain internal / blocked evidence rather than public claims.

Acceptance Criteria
-------------------

- Every claimed benchmark has a claim declaration (benchmark-claims/<name>.json).
- Every declaration has dataset source, license, command, commit, package version,
  model, scorer, and baseline.
- executionFailures = 0.
- No same-model judge for public semantic-judge claims.
- No additional MemoryAgentBench scope or LoCoMo README row promotion until the
  claim gate passes for it; current MemoryAgentBench public scope is only CR/TTL.
- BEAM always reports fitted and generalization recall together.
- The claim gate (gate:public-benchmark-claim --strict) is consistent (no
  over-claiming) before any README benchmark row changes.
