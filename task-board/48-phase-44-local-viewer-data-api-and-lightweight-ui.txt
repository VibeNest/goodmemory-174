Phase 44: Local Viewer Data API and Lightweight UI
==================================================

Status
------
[TODO] Not started.


Intent
------
Make GoodMemory observable locally without building a dashboard product. Phase
44 starts with a read-only data API and a lightweight static viewer shell.


Core Boundary
-------------
Viewer is an optional local inspection surface. It must not mutate memory, must
not expose raw transcripts, and must not introduce cloud, account, analytics, or
managed-dashboard behavior.


Accepted Scope
--------------
- read-only viewer data API
- local static viewer shell
- progressive record drill-down using Phase 42 recordRef
- memory counts, recall index, writeback audit, runtime sessions, and trace summaries
- local token and 127.0.0.1-only binding
- CLI handoff for forget/revise commands
- package/license hygiene gate


Out of Scope
------------
- dashboard/admin product
- mutation routes
- hosted sync or managed cloud
- analytics
- CORS-enabled API
- raw transcript archive
- React as a required v1 blocker


Acceptance Criteria
-------------------
1. Viewer binds `127.0.0.1` only.
2. Viewer routes require a local token or one-time session secret.
3. Viewer sends no permissive CORS headers.
4. Viewer exposes no write mutation routes.
5. Viewer never displays raw transcript content.
6. Forget/revise actions generate CLI handoff commands but do not execute them.
7. Packed tarball excludes `third-party/claude-mem-main`.
