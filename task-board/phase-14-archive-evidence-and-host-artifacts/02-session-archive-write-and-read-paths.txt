Phase 14.2: Session Archive Write and Read Paths
================================================

[DONE] P14.2-T001 Add session archive repository/storage support
[DONE] P14.2-T002 Persist archive records from runtime/session-close or maintenance flows
[DONE] P14.2-T003 Add scope-aware archive retrieval for continuation queries

Why this exists
---------------
- `EpisodeMemory` is not enough to answer "what happened last time" across longer lifecycles.
- Archive recall should improve continuity before embeddings or reranking are introduced.

Write tests first
-----------------
- archive writes are isolated by user/workspace/tenant scope
- archive reads do not leak across adjacent scopes
- continuation-style recalls can return archive-backed results when durable facts are insufficient

Implementation steps
--------------------
1. Add archive persistence hooks behind existing runtime/maintenance seams
2. Store normalized and summarized session continuity data without promoting it to durable fact truth
3. Expose archive retrieval through recall planning inputs, not through a new public API surface

Acceptance
----------
- Cross-session continuity has a first-class substrate
- Archive recall works without provider-backed retrieval


Implemented
-----------
- Added archive repository/storage support through `src/storage/repositories.ts`
- Added optional runtime archive persistence on `endSession()` in `src/runtime/contextService.ts`
- Archive writes stay optional and no-op for untouched empty sessions
- Persisted archive fields now include summary, normalized transcript, key decisions, unresolved items, referenced artifacts, and scope lineage
- Added scope-aware archive retrieval for continuation queries in `src/recall/engine.ts`
- Added archive summaries to recall packets and markdown context output
- Added archive continuity integration tests that exercise runtime -> archive -> recall end-to-end


Completed in this slice
-----------------------
- maintenance consolidation now salvages archived episodes into session archives when no archive already exists for the source session
- maintenance archive salvage stays scope-aware and avoids duplicate archive creation for sessions that already have an archive
