Phase 30: Trace-Backed Behavioral Enactment and Live Closure
============================================================

Phase Goal
----------
Close the remaining gap between Phase 25 deterministic behavioral evidence and real agent behavior by making behavioral scoring consume the first executable host/runtime trace event, not model self-reported `first_action` JSON.

This phase proves that GoodMemory can change the first real action taken by the accepted Codex coding-agent path, and that provider-backed live-memory behavioral closure is accepted as evidence.

Definition of Done
------------------
- coding-agent behavioral cases score from the first executable trace event only
- later actions, model explanations, and self-corrections cannot repair a failed first action score
- deterministic trace fixtures prove procedural generalization, conditioning avoidance, timeout aversion, explicit user correction, and safe-warning behavior
- failed first actions from host traces create `tool_outcome` telemetry with evidence lineage
- repeated host-trace failures produce accepted `procedural_pattern` proposals and compile into active `validated_pattern` feedback
- provider-backed live-memory Phase 30 report is archived and accepted before `gate:phase-30` can pass
- Phase 25 `layer_d` metric names remain unchanged
- no public `GoodMemory` API widening, no README-level default behavior change, and no reopening of the Phase 29 release boundary

Scope Boundary
--------------
- This phase upgrades behavioral evidence from self-reported action envelopes to trace-backed first actions.
- This phase does not add a new public durable memory type; continue to reuse `validated_pattern`.
- This phase does not make Claude a gate-blocking host path.
- This phase does not add installer automation, publish automation, or a new release workflow.
- This phase does not change the accepted Phase 28 local backend contract or Phase 29 Bun-only RC release boundary.


Implemented So Far
------------------
- internal Codex-style behavioral trace contract now exists under `src/host/behavioralTrace.ts`
- deterministic behavioral scoring now prefers the first executable trace event over self-reported `first_action`
- later repaired actions in the same trace do not repair a failed first-action score
- failed trace-first Codex events can now be converted into internal outcome telemetry through `src/host/behavioralTraceBridge.ts`
- the accepted Codex host adapter now exposes an internal trace-recording helper; automatic runtime emission is still pending
- repeated failed Codex traces now promote through the existing telemetry/proposal/compiler chain into active `validated_pattern` feedback
- deterministic behavioral eval fixtures can now replay `behavioral_trace_replays` through the outcome-telemetry profile
- a dedicated Phase 30 deterministic runner now reads `fixtures/behavioral-enactment/` and writes `reports/eval/fallback/phase-30/<run-id>/report.json`
- structured cases can now require trace-backed scoring, so self-reported `first_action` JSON fails closed when a Phase 30 runner demands host traces
- per-case Phase 30 evidence now includes baseline trace, GoodMemory trace, first-action source/parse, memory context, score reason, and outcome-telemetry lineage
- Phase 25 behavioral eval regressions still pass after the trace-priority change
