Why this exists
---------------
- The repo's main product priorities are identity/background understanding and historical task/open-loop continuation; Phase 27 must prove both.
- “beats baseline” is not meaningful unless the baseline is frozen and the thresholds are explicit.
- Adoption evidence should show that GoodMemory improves real user continuity, not just that wrappers and examples compile.

Write tests first
-----------------
- baseline and GoodMemory runs share the same model, system prompt, messages/transcript, tool shape, and host artifact input
- identity/background family distinguishes actual user-memory wins from noise
- continuation/open-loop family measures blocker and open-loop carry-forward
- repeated-correction family measures whether corrections stop recurring
- Codex handoff family measures whether compiled artifacts resume the intended context

Implementation steps
--------------------
1. Define one canonical deterministic baseline contract that uses the same rule-shaped answer builder with visible-transcript-only context for baseline and memory-augmented context for GoodMemory.
2. Curate three deterministic scenario families on top of the existing eval dataset:
   - 3 identity/background scenarios
   - 6 continuation/open-loop scenarios
   - 4 repeated-correction scenarios
3. Add one deterministic Codex handoff/resume family with 3 file-assisted cases.
4. Emit thresholded Phase 27 metrics and fail the report when any metric is below threshold.

Acceptance
----------
- identity/background: GoodMemory wins at least 2 of 3 cases
- continuation/open-loop: GoodMemory has at least 2 net wins and baseline wins at most 1 case
- repeated-correction: GoodMemory improves repeated-correction rate by at least 25 percentage points
- Codex handoff/resume: all 3 cases pass
- setup-surface and public-reference-purity metrics remain green in the same deterministic gate
