Phase 30.1: Trace Contract and Source of Truth
==============================================

[DONE] Define one internal behavioral trace event schema.
[DONE] Capture `tool_call`, `command`, and `warning` first actions with normalized `name`, `args`, and `raw` fields.
[DONE] Capture outcome metadata: `success`, `failure`, `timeout`, `user_corrected`, and optional correction lineage.
[DONE] Add parser/extractor tests proving the first executable trace event is the only scoring source.

Acceptance
----------
- Trace extraction ignores later actions and explanations.
- Each trace event uses a unique non-negative integer `stepIndex` so the first executable action has a total order.
- Same-action normalization is stable across raw command text and structured arguments.
- Invalid, missing, or ambiguous first-action trace data fails closed instead of falling back to model prose.
