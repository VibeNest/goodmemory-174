Phase 27: Reference Integration Gate and Adoption Evidence
==========================================================

Phase Goal
----------
Prove that GoodMemory's public reference path is adoptable and measurably useful on top of the accepted Phase 26 local-first runtime.

This phase exists to harden and verify:

- the public `goodmemory` + `goodmemory/ai-sdk` reference integration path
- the public `goodmemory/host` Codex file-assisted handoff/resume path
- deterministic and live adoption evidence for user-memory value

This phase does not add new memory capability. It proves that the current public surface can be adopted through the stable local-first runtime shape that Phase 26 closed.

Definition of Done
------------------
- the canonical AI SDK reference path uses `createGoodMemory({})` and the stable Phase 26 auto-storage default rather than `storage: { provider: "memory" }`
- the canonical deterministic reference path runs with local SQLite default storage, no embedding env, and explicit `rules-only` behavior
- gated docs/examples/tests import only:
  - `goodmemory`
  - `goodmemory/ai-sdk`
  - `goodmemory/host`
- no gated doc/example/test imports `../src/*` or `../../src/*`
- the package-boundary consumer smoke fixture runs through package-name imports instead of repo-internal source paths
- deterministic adoption eval freezes one baseline contract:
  - same model
  - same system prompt
  - same messages/transcript
  - same tool shape
  - same host artifact input when relevant
  - only memory augmentation differs
- deterministic families include at least:
  - 3 identity/background cases
  - 6 continuation/open-loop cases
  - 4 repeated-correction cases
  - 3 Codex handoff/resume cases
- deterministic acceptance thresholds are:
  - identity/background: GoodMemory wins at least 2 of 3 cases
  - continuation/open-loop: GoodMemory has at least 2 net wins and baseline wins at most 1 case
  - repeated-correction: GoodMemory improves repeated-correction rate by at least 25 percentage points
  - Codex handoff/resume: 3 of 3 cases pass
  - setup and public-surface purity both pass
- at least 1 provider-backed live run is archived for Phase 27
- the live run covers continuation/open-loop and repeated-correction families and shows a strict majority of GoodMemory wins, with baseline winning at most 1 case
- Codex is the only gate-blocking host path in this phase; Claude remains docs/example coverage only
- no installer CLI, no `src/core` refactor, no package publish automation, and no new memory capability work lands inside this phase
