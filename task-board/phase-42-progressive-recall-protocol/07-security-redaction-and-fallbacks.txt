Phase 42.7: Security, Redaction, and Fallbacks
==============================================

Status
------
[DONE] Accepted as part of the Phase 42 progressive recall protocol closure.

Goal
----
Protect progressive recall from scope leaks and privacy regressions.

Required Tests
--------------
- cross-scope recordRef denial
- `remember: "never"` redaction
- private/session-only exclusion
- sensitive candidate suppression
- MCP unavailable fallback
- malformed recordRef fail-closed

Acceptance
----------
- Progressive index and detail output are redaction-safe by default.
- ScopeDigest is the only scope identifier exposed to the model.
- Fallback never makes recall weaker than the current fragment path.

-----------------
- redaction and cross-scope tests in `tests/unit/progressive-recall.service.test.ts`
- MCP cache fallback current-scope denial in `tests/integration/host-mcp-server.test.ts`
- installed-host no-MCP fallback in `tests/unit/host-hook-runtime.test.ts`
- deterministic eval: `reports/eval/fallback/phase-42/run-20260426093000/report.json`
