Phase 73: Codex Installed-Host Coding Effect Evaluation
=======================================================

Status
------

[ACTIVE]

Goal
----

Prove whether enabling the packaged GoodMemory installed-host path improves
real Codex coding outcomes across fresh sessions, using paired repository
episodes and evaluator-owned hidden tests.

Gate
----

- The primary metric is hidden-test resolve@1 for stage positions 2+, with the
  same prepared snapshot, prompt and prompt hash, Codex executable hash and
  version, model, reasoning effort, sandbox, budget, and repository-instruction
  hashes across paired arms. Arm-specific host configuration is the treatment:
  its normalized diff must be frozen and persisted rather than falsely requiring
  the complete host configurations to be identical.
- C3's installed arm runs the packaged
  `goodmemory setup --recommended --host codex --user-id <id> --yes --json`
  surface in an isolated home with global activation. It does not run workspace
  `goodmemory enable`, and setup must not create task-workspace `.goodmemory`
  state or alter repository instructions.
- Public status must resolve global activation and workspace status as healthy
  and bind
  the `coding_agent` / `selective` profile, raw transcript persistence disabled,
  and registered native hooks plus MCP.
- Public evidence requires the accepted full gate. Host canaries, deterministic
  smoke, pilots, MemGym CodeQA, memory QA, and retrieval diagnostics are not
  public coding-effect proof.
- The first accepted claim is Codex-only. Claude Code remains deferred.

--------

`tests/`, fixtures, and report directories. Production `src/` changes require a
reproduced native-host defect or a genuine public product observability need.
