Phase 43.5: Optional Runtime Worker
===================================

Status
------
[DONE] Accepted as the Phase 43.5 Optional Runtime Worker slice.


Intent
------
Add an optional worker layer for bounded runtime-kit jobs without making daemon,
sidecar, queue, or background processing part of GoodMemory core.


Core Boundary
-------------
The worker consumes runtime-kit job envelopes. It must not become the canonical
memory path, must not persist raw transcripts, and must not block inline runtime
behavior.


Accepted Scope
--------------
- bounded job envelope
- coalescing and idempotency rules
- `drain-once`, `status`, and `recover` commands
- dry-run recovery before mutation
- auditable stuck/retry state
- optional daemon `start`/`stop` as an enhancement, not a Phase 43 blocker


Out of Scope
------------
- persistent distributed job queue
- managed worker service
- cloud sync
- raw transcript archive
- making daemon mode required for installed-host recall/writeback/pre-action


Acceptance Criteria
-------------------
1. Worker jobs contain no raw transcript or full assistant output.
2. `drain-once`, `status`, and `recover --dry-run` close before daemon mode is
   required.
3. Worker failure cannot break inline recall, writeback, or pre-action.
4. Stuck/retry/recovery state is inspectable and auditable.
5. Coalescing prevents repeated equivalent jobs from creating duplicate writes.
